{<P><b>There is a defect related to time zones.</b> 
Example: if you are in Ottawa, then you can view data for Washington DC, since those two cities are in the same time zone.
If you are in Ottawa and you view data for Los Angeles, however, then the times are going to be messed up.
<em>Everything works fine only if you stay in the same time zone used by your browser.</em>
Most of the time, this defect won't bother you, because you're usually concerned with nearby locations.
(This problem is caused by how Javascript treats dates and times.
There are ways around this problem, but I haven't done anything about it yet. 
If you need a work-around, then just temporarily change your computer's time zone setting.)}

{<P><b>Il y a un problème avec les les fuseaux horaires.</b> 
Par exemple: si on se trouve à Ottawa, on peut voir les données correctes pour Washington DC, car les deux villes se trouvent dans le mème fuseau horaire.
Toutefois, si on se trouve à Ottawa, on ne peut pas voir les données correctes pour Los Angeles: les temps ne sont pas correctes.
<em>Tout va bien seulement si on reste dans le mème fuseau horaire emloyé par votre navigateur.</em>
La plupart du temps, ce problème ne va pas vous déranger, car les locations proches sont les plus intéressantes.
(La cause de ce problème est le design du langage Javascript.
Il y a des méthodes pour résoudre ce problème, mais, pour le moment, il est toujours dans le système. 
Si il faut voir les données correctes pour une location dans un autre fuseau horaire, vous pouvez toujours changer le fuseau horaire de votre navigateur.)}
